# Day 1 - Project Setup and Planning

## Project

**Student Homework Tracker**

## Date

Day 1 of the Student Homework Tracker project focused on planning the project properly before starting the main build.

The goal for this day was to create a realistic project workflow using Trello and GitHub, so the project could be documented from the beginning and show a clear development process.

---

## Main Goal For Day 1

The main goal was to set up the project foundation.

This included:

- Creating the project idea
- Planning the main features
- Setting up a Trello board
- Creating a GitHub repository
- Adding project documentation
- Preparing the project for future coding work

---

## Project Idea

The project is a student dashboard that helps students track homework tasks.

Each homework task should include:

- Subject
- Task title
- Due date
- Priority
- Completion status
- Overdue status

The project is designed to solve a real-world student problem: keeping track of homework from different subjects in one organized place.

---

## Trello Planning

A Trello board was created to plan and manage the project.

The board was organized into realistic project lists:

- Project Brief
- Ideas / Extras
- To Do
- In Progress
- Review / Testing
- Bugs / Problems
- Done

The Trello board helps organize the project into smaller tasks and gives the project a more realistic development workflow.

---

## Trello Board Setup

The first Trello board included cards for:

- Target user
- Future feature ideas
- Page layout planning

- CSS layout
- Homework form design
- Homework display area
